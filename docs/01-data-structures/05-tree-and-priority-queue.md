# 트리(Tree)와 우선순위 큐(Priority Queue)

## 학습 목표

- 트리의 기본 용어와 계층 구조를 설명한다.
- 이진 트리와 균형의 중요성을 이해한다.
- 최소 힙 기반 우선순위 큐를 JavaScript로 구현한다.

## 트리

트리는 부모와 자식 관계로 계층을 표현하는 비선형 자료구조다. 파일 시스템, 조직도, DOM처럼 계층적인 데이터를 나타낼 때 사용한다.

```text
          A         ← 루트
        /   \
       B     C
      / \     \
     D   E     F    ← 리프
```

### 기본 용어

위 예시 트리를 기준으로 용어를 정리하면 다음과 같다.

- **루트 노드(root)**: 부모가 없는 최상위 노드다.
  - ex. `A`
- **부모 노드(parent)**: 다른 노드로 내려가는 간선을 가진 바로 위 노드다.
  - ex. `B`는 `D`, `E`의 부모 노드
- **자식 노드(child)**: 부모 노드 바로 아래에 연결된 노드다.
  - ex. `D`, `E`는 `B`의 자식 노드
- **형제 노드(sibling)**: 같은 부모를 가진 노드다.
  - ex. `B`와 `C`는 부모가 `A`로 같으므로 형제 노드
- **리프 노드(leaf)**: 자식이 없는 노드다.
  - ex. `D`, `E`, `F`
- **깊이(depth)**: 루트에서 특정 노드까지 내려갈 때 지나는 간선의 수다. 루트의 깊이는 `0`이다.
  - ex. `A → B → D`로 이동하면 간선이 `2`개이므로 `D`의 깊이는 `2`
- **레벨(level)**: 루트부터 몇 번째 층에 있는지를 나타낸다. 보통 루트를 `1`레벨로 본다.
  - ex. `A`는 `1`레벨, `B`와 `C`는 `2`레벨, `D`, `E`, `F`는 `3`레벨
- **높이(height)**: 특정 노드에서 가장 먼 리프까지 내려갈 때 지나는 간선의 수다. 리프 노드의 높이는 `0`이다.
  - ex. `A → B → D` 또는 `A → C → F`로 이동하면 간선이 `2`개이므로 `A`의 높이는 `2`
- **서브트리(subtree)**: 특정 노드와 그 자손들로 이루어진 작은 트리다.
  - ex. `B`를 루트처럼 보면 `B`, `D`, `E`로 이루어진 부분이 하나의 서브트리

## 이진 트리와 균형

이진 트리는 각 노드가 최대 두 개의 자식을 갖는 트리다. 이진 탐색 트리에서는 일반적으로 왼쪽 서브트리에 더 작은 값, 오른쪽 서브트리에 더 큰 값을 둔다.

```text
        8
      /   \
     3     10
    / \      \
   1   6      14
      / \     /
     4   7   13
```

위 예시는 이진 탐색 트리 형태다. `8`을 기준으로 왼쪽에는 `8`보다 작은 값들이 있고, 오른쪽에는 `8`보다 큰 값들이 있다. 또한 각 노드는 왼쪽 자식과 오른쪽 자식을 합쳐 최대 두 개의 자식만 가진다.

### 이진 트리의 종류

이진 트리는 노드가 채워진 형태에 따라 여러 종류로 나눌 수 있다. 특히 힙을 이해하려면 **완전 이진 트리**의 개념을 알고 있어야 한다.

#### 포화 이진 트리

포화 이진 트리(full binary tree)는 모든 내부 노드가 정확히 두 개의 자식을 가지는 이진 트리다. 즉, 자식이 있다면 반드시 왼쪽과 오른쪽 자식이 모두 있어야 한다.

```text
        A
      /   \
     B     C
    / \   / \
   D   E F   G
```

위 예시에서 `A`, `B`, `C`는 모두 자식을 두 개씩 가진다. `D`, `E`, `F`, `G`는 자식이 없는 리프 노드다. 따라서 포화 이진 트리다.

반대로 아래처럼 어떤 노드가 자식을 하나만 가지면 포화 이진 트리가 아니다.

```text
        A
      /   \
     B     C
    /
   D
```

`B`가 왼쪽 자식 `D`만 가지고 오른쪽 자식이 없기 때문이다.

#### 완전 이진 트리

완전 이진 트리(complete binary tree)는 마지막 레벨을 제외한 모든 레벨이 채워져 있고, 마지막 레벨의 노드가 왼쪽부터 차례대로 채워진 이진 트리다.

```text
        A
      /   \
     B     C
    / \   /
   D   E F
```

위 예시는 마지막 레벨에 `D`, `E`, `F`가 왼쪽부터 채워져 있으므로 완전 이진 트리다. 마지막 레벨이 전부 꽉 차 있지 않아도 괜찮지만, 중간에 빈자리가 생기면 안 된다.

아래 예시는 완전 이진 트리가 아니다.

```text
        A
      /   \
     B     C
      \     \
       E     G
```

마지막 레벨에서 왼쪽부터 채워지지 않고 빈자리를 건너뛰었기 때문이다. 힙은 이 **완전 이진 트리 구조**를 기반으로 구현된다.

#### 높이 균형 트리

높이 균형 트리(height-balanced tree)는 왼쪽 서브트리와 오른쪽 서브트리의 높이 차이가 크게 벌어지지 않도록 유지하는 트리다. 일반적으로 각 노드에서 왼쪽과 오른쪽 서브트리의 높이 차이가 `1` 이하인 경우를 균형 잡힌 상태로 본다.

```text
        8
      /   \
     4     12
    / \    / \
   2   6 10  14
```

위 트리는 루트를 기준으로 왼쪽과 오른쪽의 높이가 비슷하다. 이런 형태에서는 탐색할 때 비교할 때마다 탐색 범위가 절반 가까이 줄어들 수 있다.

반대로 아래처럼 한쪽으로 치우친 트리는 높이 균형 트리가 아니다.

```text
    1
     \
      2
       \
        3
         \
          4
```

이 경우 사실상 연결 리스트처럼 동작하므로 탐색 성능이 `O(log n)`이 아니라 `O(n)`에 가까워질 수 있다.

트리가 한쪽으로 치우치면 높이가 `n`에 가까워져 탐색이 `O(n)`까지 느려질 수 있다. 반대로 높이가 균형을 이루면 탐색 범위가 단계마다 줄어들어 탐색, 삽입, 삭제를 `O(log n)`에 수행할 수 있다.

## 우선순위 큐

우선순위 큐는 먼저 들어온 순서가 아니라 **우선순위가 높은 원소를 먼저 꺼내는** 자료구조다.

- 최소 우선순위 큐: 값이 작은 원소부터 추출
- 최대 우선순위 큐: 값이 큰 원소부터 추출

일반 큐는 삽입된 순서를 기준으로 동작하지만, 우선순위 큐는 각 데이터가 가진 우선순위를 기준으로 동작한다. 예를 들어 병원 응급실에서는 먼저 온 사람보다 더 위급한 환자가 먼저 진료를 받을 수 있다. 이런 구조를 코드로 표현한 것이 우선순위 큐다.

### 우선순위 큐를 구현하는 방법

우선순위 큐는 여러 방식으로 만들 수 있다.

| 구현 방식 | 삽입 | 삭제 | 특징 |
| --- | --- | --- | --- |
| 정렬되지 않은 배열 | `O(1)` | `O(n)` | 넣기는 쉽지만, 매번 우선순위가 가장 높은 값을 찾아야 한다. |
| 정렬된 배열 | `O(n)` | `O(1)` | 꺼내기는 쉽지만, 넣을 때 정렬 위치를 찾아야 한다. |
| 힙(heap) | `O(log n)` | `O(log n)` | 삽입과 삭제 모두 안정적으로 빠르다. |

코딩 테스트에서는 데이터가 많이 들어오고 자주 빠져나가는 경우가 많다. 그래서 우선순위 큐는 보통 **힙**으로 구현한다.

## 힙

힙은 완전 이진 트리를 기반으로 하는 자료구조다. 완전 이진 트리는 마지막 레벨을 제외하면 모든 레벨이 채워져 있고, 마지막 레벨은 왼쪽부터 채워지는 형태다.

```text
        1
      /   \
     3     5
    / \   /
   7   9 8
```

위 트리는 마지막 줄이 왼쪽부터 채워져 있으므로 완전 이진 트리 형태다. 힙은 이 구조를 유지하면서 부모와 자식 사이에 우선순위 규칙을 둔다.

- **최소 힙(min heap)**: 부모 노드의 값이 자식 노드의 값보다 작거나 같다. 루트에는 최솟값이 온다.
- **최대 힙(max heap)**: 부모 노드의 값이 자식 노드의 값보다 크거나 같다. 루트에는 최댓값이 온다.

### 최소 힙 예시

```text
        1
      /   \
     3     5
    / \   /
   7   9 8
```

모든 부모가 자식보다 작거나 같다.

- `1 <= 3`, `1 <= 5`
- `3 <= 7`, `3 <= 9`
- `5 <= 8`

따라서 루트 노드 `1`이 전체에서 가장 작은 값이다.

### 최대 힙 예시

```text
        9
      /   \
     7     8
    / \   /
   1   3 5
```

모든 부모가 자식보다 크거나 같다.

- `9 >= 7`, `9 >= 8`
- `7 >= 1`, `7 >= 3`
- `8 >= 5`

따라서 루트 노드 `9`가 전체에서 가장 큰 값이다.

### 힙을 배열로 표현하기

완전 이진 트리는 배열로 효율적으로 표현할 수 있다. 인덱스 `i`를 기준으로 관계는 다음과 같다.

- 부모: `Math.floor((i - 1) / 2)`
- 왼쪽 자식: `2 * i + 1`
- 오른쪽 자식: `2 * i + 2`

예를 들어 다음 최소 힙을 배열로 나타내면 `[1, 3, 5, 7, 9, 8]`이 된다.

```text
인덱스:  0  1  2  3  4  5
배열:   [1, 3, 5, 7, 9, 8]

        1(0)
       /    \
    3(1)    5(2)
    /  \     /
 7(3) 9(4) 8(5)
```

배열을 사용하면 실제 노드 객체를 연결하지 않아도 부모와 자식 위치를 계산할 수 있다. 이 점 때문에 힙은 보통 배열 하나로 구현한다.

### 힙의 삽입 과정

최소 힙에 새 값을 넣을 때는 다음 순서로 동작한다.

1. 새 값을 배열의 마지막에 넣는다.
2. 부모와 비교한다.
3. 부모보다 작으면 부모와 위치를 바꾼다.
4. 힙 조건을 만족할 때까지 위로 올라간다.

이 과정을 보통 `bubble up`, `sift up`, `heapify up`이라고 부른다.

```text
기존 힙: [1, 3, 5, 7, 9, 8]
삽입 값: 2

1. 마지막에 추가

        1
      /   \
     3     5
    / \   / \
   7   9 8   2

2. 부모 5보다 작으므로 교환

        1
      /   \
     3     2
    / \   / \
   7   9 8   5
```

이제 부모 `1`이 `2`보다 작으므로 멈춘다.

### 힙의 삭제 과정

최소 힙에서 값을 꺼낼 때는 루트, 즉 최솟값을 삭제한다.

1. 루트 값을 결과로 저장한다.
2. 마지막 값을 루트 자리로 옮긴다.
3. 자식 중 더 작은 값과 비교한다.
4. 자식보다 크면 위치를 바꾼다.
5. 힙 조건을 만족할 때까지 아래로 내려간다.

이 과정을 보통 `bubble down`, `sift down`, `heapify down`이라고 부른다.

```text
기존 힙: [1, 3, 2, 7, 9, 8, 5]
삭제 값: 1

1. 마지막 값 5를 루트로 이동

        5
      /   \
     3     2
    / \   /
   7   9 8

2. 자식 중 더 작은 값 2와 교환

        2
      /   \
     3     5
    / \   /
   7   9 8
```

이제 `5`는 자식 `8`보다 작으므로 멈춘다.

힙에서 삽입과 삭제를 할 때는 트리의 한 경로를 따라 위나 아래로 이동한다. 한 단계 이동할 때마다 고려해야 하는 범위가 절반씩 줄어드는 셈이므로, 삽입과 삭제의 시간 복잡도는 `O(log n)`을 보장한다.

## 최소 힙 구현

JavaScript 표준 내장 객체에는 범용 우선순위 큐나 힙이 없다. 따라서 코딩 테스트에서는 직접 구현하는 경우가 많고, 실제 프로젝트에서는 필요에 따라 별도 라이브러리를 사용할 수 있다.

JavaScript에서 사용할 수 있는 우선순위 큐 라이브러리 예시로는 [`priorityqueuejs`](https://github.com/ndb796/priorityqueuejs)가 있다. 다만 코딩 테스트 환경에서는 외부 라이브러리를 사용할 수 없는 경우가 많으므로, 최소 힙 구현 방식을 직접 익혀두는 편이 좋다.

```js
class MinHeap {
  #heap = []

  get size() {
    return this.#heap.length
  }

  peek() {
    return this.#heap[0]
  }

  push(value) {
    this.#heap.push(value)
    this.#bubbleUp()
  }

  pop() {
    if (this.size === 0) return undefined
    if (this.size === 1) return this.#heap.pop()

    const min = this.#heap[0]
    this.#heap[0] = this.#heap.pop()
    this.#bubbleDown()
    return min
  }

  #bubbleUp() {
    let index = this.size - 1

    while (index > 0) {
      const parent = Math.floor((index - 1) / 2)
      if (this.#heap[parent] <= this.#heap[index]) break

      ;[this.#heap[parent], this.#heap[index]] = [
        this.#heap[index],
        this.#heap[parent]
      ]
      index = parent
    }
  }

  #bubbleDown() {
    let index = 0

    while (true) {
      const left = index * 2 + 1
      const right = index * 2 + 2
      let smallest = index

      if (left < this.size && this.#heap[left] < this.#heap[smallest]) {
        smallest = left
      }
      if (right < this.size && this.#heap[right] < this.#heap[smallest]) {
        smallest = right
      }
      if (smallest === index) break

      ;[this.#heap[index], this.#heap[smallest]] = [
        this.#heap[smallest],
        this.#heap[index]
      ]
      index = smallest
    }
  }
}

const heap = new MinHeap()
for (const number of [5, 3, 8, 1]) heap.push(number)

console.log(heap.pop()) // 1
console.log(heap.pop()) // 3
```

## 복잡도 분석

| 연산 | 시간 복잡도 | 이유 |
| --- | --- | --- |
| 최솟값 확인 | `O(1)` | 최소 힙에서는 루트가 항상 최솟값이다. |
| 삽입 | `O(log n)` | 마지막에 넣고 트리 높이만큼 위로 이동한다. |
| 최솟값 삭제 | `O(log n)` | 루트를 제거한 뒤 트리 높이만큼 아래로 이동한다. |
| 전체 공간 | `O(n)` | 모든 원소를 배열에 저장한다. |

## 활용 사례

- 다익스트라 최단 경로 알고리즘
- 우선순위가 있는 작업 스케줄링
- 가장 크거나 작은 `k`개의 값 찾기
- 여러 정렬된 데이터 병합
