# 자료구조(Data Structure)

## 학습 목표

- 자료구조가 필요한 이유를 설명할 수 있다.
- 문제의 특성에 맞는 자료구조를 선택해야 하는 이유를 이해한다.
- 주요 연산의 시간 복잡도를 비교하는 기준을 익힌다.

## 핵심 개념

자료구조는 여러 데이터를 저장하고, 필요한 연산을 효율적으로 수행하기 위한 구조다. 데이터가 적을 때는 구조의 차이가 잘 드러나지 않지만 데이터의 수가 커질수록 조회, 삽입, 삭제 방식에 따른 성능 차이가 커진다.

자료구조와 알고리즘은 서로 분리된 개념이 아니다.

- 자료구조를 구현하려면 삽입, 삭제, 탐색을 수행하는 알고리즘이 필요하다.
- 알고리즘 문제를 효율적으로 해결하려면 문제에 맞는 자료구조를 선택해야 한다.

예를 들어 가장 최근에 들어온 값을 먼저 처리해야 한다면 스택이 자연스럽고, 먼저 들어온 값을 먼저 처리해야 한다면 큐가 자연스럽다.

## 자료구조의 종류

### 선형 자료구조

데이터가 앞뒤 관계를 가지며 일렬로 연결되는 구조다.

```text
A → B → C → D
```

대표적으로 배열, 연결 리스트, 스택, 큐, 덱이 있다.

### 비선형 자료구조

하나의 데이터가 여러 데이터와 연결될 수 있는 구조다.

```text
        A
      /   \
     B     C
    / \     \
   D   E     F
```

대표적으로 트리와 그래프가 있다.

## 프로그램의 성능 측정

자료구조와 알고리즘의 효율은 보통 두 기준으로 평가한다.

### 시간 복잡도

입력 크기 `n`이 증가할 때 필요한 연산 횟수가 어떤 비율로 증가하는지를 나타낸다. 실제 실행 시간은 하드웨어와 실행 환경에 영향을 받으므로, 보통 연산 횟수의 증가율을 비교한다.

### 공간 복잡도

입력 크기 `n`이 증가할 때 알고리즘이 추가로 사용하는 메모리의 증가율을 나타낸다. 시간을 줄이기 위해 메모리를 더 사용하는 방식도 흔하다. 중복 계산 결과를 저장하는 메모이제이션이 대표적인 예다.

## Big-O 표기법

Big-O 표기법은 입력 크기가 충분히 커졌을 때 가장 영향력이 큰 증가 항에 집중한다.

```text
3n² + 5n + 10  →  O(n²)
```

상수 계수와 낮은 차수의 항을 제거하면 `n²`이 남기 때문이다.

| 표기 | 이름 | 입력이 커질 때의 경향 | 예시 |
| --- | --- | --- | --- |
| `O(1)` | 상수 시간 | 입력 크기와 무관 | 배열 인덱스 접근 |
| `O(log n)` | 로그 시간 | 탐색 범위가 반복해서 줄어듦 | 이진 탐색, 힙 연산 |
| `O(n)` | 선형 시간 | 입력에 비례 | 배열 전체 순회 |
| `O(n log n)` | 선형 로그 시간 | 효율적인 비교 정렬의 전형 | 병합 정렬 |
| `O(n²)` | 제곱 시간 | 중첩 반복이 많음 | 단순한 이중 반복 |

::: warning Big-O를 볼 때 주의할 점
Big-O는 증가율을 나타낸다. 같은 `O(n)` 알고리즘이라도 실제 연산 수와 실행 환경에 따라 속도는 다를 수 있다.
:::

## 간단한 복잡도 예시

```js
const numbers = [10, 20, 30, 40]

// 인덱스를 알고 있으므로 O(1)
console.log(numbers[2])

// 모든 원소를 확인하므로 O(n)
for (const number of numbers) {
  console.log(number)
}

// 모든 원소 쌍을 확인하므로 O(n²)
for (const left of numbers) {
  for (const right of numbers) {
    console.log(left, right)
  }
}
```

## 자료구조를 선택하는 질문

1. 어떤 연산이 가장 자주 발생하는가?
2. 데이터의 순서를 유지해야 하는가?
3. 중복 데이터를 허용하는가?
4. 삽입과 삭제는 어느 위치에서 일어나는가?
5. 데이터의 최대 크기는 어느 정도인가?

## 복습 질문

1. 자료구조와 알고리즘의 차이는 무엇인가?
2. 동일한 데이터를 서로 다른 자료구조로 관리하면 어떤 차이가 생기는가?
3. `O(n² + n)`을 Big-O로 표현하면 무엇인가?
4. 시간을 줄이기 위해 공간을 더 사용하는 예에는 무엇이 있는가?
