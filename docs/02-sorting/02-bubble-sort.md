# 버블 정렬(Bubble Sort)

## 버블 정렬이란?

버블 정렬은 **인접한 두 원소를 비교**해서 정렬 순서가 맞지 않으면 서로 위치를 바꾸는 정렬 방법이다.

서로 인접한 두 원소를 계속 비교하고 교환하는 모습이 거품이 올라가는 것과 비슷하다고 해서 버블 정렬이라고 부른다.

핵심 아이디어는 다음과 같다.

1. 첫 번째 원소와 두 번째 원소를 비교한다.
2. 두 번째 원소와 세 번째 원소를 비교한다.
3. 세 번째 원소와 네 번째 원소를 비교한다.
4. 필요한 경우 위치를 교환한다.
5. 한 단계를 마치면 가장 큰 값이 맨 뒤로 이동한다.

버블 정렬은 구현이 단순하지만 시간 복잡도가 `O(N²)`이므로 비효율적인 정렬 알고리즘 중 하나다.

## 동작 방식

다음 배열을 오름차순으로 정렬해보자.

```text
[7, 5, 9, 0, 3]
```

버블 정렬은 왼쪽부터 인접한 두 원소를 비교한다. 왼쪽 값이 오른쪽 값보다 크면 두 값을 교환한다.

### 1단계

처음에는 전체 배열을 왼쪽부터 오른쪽까지 확인한다.

```text
초기 상태: [7, 5, 9, 0, 3]
```

`7`과 `5`를 비교한다. `7 > 5`이므로 교환한다.

```text
[7, 5, 9, 0, 3]
 ↓  ↓
[5, 7, 9, 0, 3]
```

`7`과 `9`를 비교한다. 이미 `7 < 9`이므로 그대로 둔다.

```text
[5, 7, 9, 0, 3]
    ↓  ↓
[5, 7, 9, 0, 3]
```

`9`와 `0`을 비교한다. `9 > 0`이므로 교환한다.

```text
[5, 7, 9, 0, 3]
       ↓  ↓
[5, 7, 0, 9, 3]
```

`9`와 `3`을 비교한다. `9 > 3`이므로 교환한다.

```text
[5, 7, 0, 9, 3]
          ↓  ↓
[5, 7, 0, 3, 9]
             ↑
           정렬 완료
```

1단계가 끝나면 가장 큰 값 `9`가 맨 뒤로 이동한다. 따라서 마지막 원소는 더 이상 확인하지 않아도 된다.

### 2단계

이제 마지막 원소 `9`를 제외한 구간만 확인한다.

```text
처리 전: [5, 7, 0, 3, 9]
          └── 미정렬 ──┘  ↑
                       정렬 완료
```

인접한 원소를 비교하며 필요한 경우 교환한다.

```text
[5, 7, 0, 3, 9]
 ↓  ↓
[5, 7, 0, 3, 9]

[5, 7, 0, 3, 9]
    ↓  ↓
[5, 0, 7, 3, 9]

[5, 0, 7, 3, 9]
       ↓  ↓
[5, 0, 3, 7, 9]
          ↑  ↑
        정렬 완료
```

2단계가 끝나면 두 번째로 큰 값 `7`이 자기 위치로 이동한다.

이 과정을 반복하면 배열은 오름차순으로 정렬된다.

```text
[0, 3, 5, 7, 9]
```

::: tip 핵심
버블 정렬은 각 단계를 거칠 때마다 가장 큰 값을 하나씩 확실하게 뒤쪽에 배치하는 방식으로 이해할 수 있다.
:::

## JavaScript 구현

```js
function bubbleSort(array) {
  for (let i = 0; i < array.length - 1; i++) {
    for (let j = 0; j < array.length - i - 1; j++) {
      if (array[j] > array[j + 1]) {
        ;[array[j], array[j + 1]] = [array[j + 1], array[j]]
      }
    }
  }

  return array
}

const numbers = [7, 5, 9, 0, 3]
console.log(bubbleSort(numbers))
// [0, 3, 5, 7, 9]
```

바깥쪽 반복문은 몇 번째 단계를 수행 중인지 나타낸다. 한 단계가 끝날 때마다 가장 큰 값이 뒤쪽에 확정되므로, 다음 단계에서는 이미 정렬된 뒤쪽 구간을 제외할 수 있다.

```js
for (let j = 0; j < array.length - i - 1; j++) {
  // ...
}
```

`array.length - i - 1`까지만 확인하는 이유는 뒤쪽 `i`개의 원소가 이미 정렬 완료된 상태이기 때문이다.

인접한 두 원소의 순서가 잘못되어 있다면 위치를 바꾼다.

```js
if (array[j] > array[j + 1]) {
  ;[array[j], array[j + 1]] = [array[j + 1], array[j]]
}
```

## 시간 복잡도

버블 정렬은 인접한 두 원소를 반복해서 비교한다.

원소가 `N`개일 때 비교 횟수는 대략 다음과 같다.

```text
(N - 1) + (N - 2) + (N - 3) + ... + 1
```

따라서 시간 복잡도는 `O(N²)`이다.

| 구분 | 시간 복잡도 |
| --- | --- |
| 최선 | `O(N²)` |
| 평균 | `O(N²)` |
| 최악 | `O(N²)` |

이 구현은 이미 정렬된 배열이라도 인접한 원소들을 계속 비교한다. 따라서 이미 정렬된 배열에 대해서도 모든 비교가 필요하며, 굉장히 비효율적인 정렬 알고리즘 중 하나에 속한다.

## 정리

- 버블 정렬은 인접한 두 원소를 비교하며 필요하면 교환한다.
- 각 단계를 거칠 때마다 가장 큰 값이 하나씩 뒤쪽에 확정된다.
- 구현은 단순하지만 시간 복잡도는 `O(N²)`이다.
- 이미 정렬된 배열에 대해서도 비교를 계속 수행하므로 비효율적이다.
